# Bedrock — SaaS 平台基座技术方案

> 版本：v0.6  日期：2026-06-09
> **定位**：多租户 SaaS 平台基座（**Bedrock**），只提供 SaaS 必备的**系统能力**（租户/套餐/RBAC/菜单/鉴权/多租户隔离）；具体租户业务功能由上层**业务模块**扩展，不属于基座。
> 本文件仅含**技术选型与架构**。系统功能设计见 [功能设计.md](功能设计.md)，开发计划与进度见 [开发进度.md](开发进度.md)。
> 技术栈：Vue3 前端 + Spring Boot 后端 + PostgreSQL（MyBatis-Plus 多租户）+ Sa-token 鉴权

---

## 1. 总体架构

**Bedrock** 是多租户 SaaS 平台基座，采用**模块化单体（Modular Monolith）**：基座系统能力（`system`）与上层业务模块（`biz/*`）同仓库分包，边界清晰、部署简单。

```
┌─────────────────────────────────────────────────────┐
│                     浏览器 (SPA)                       │
│   Vue3 + Vite + TypeScript + Pinia + Vue Router       │
│   Element Plus (UI) + Axios + ECharts(数据可视化)      │
└───────────────────────────┬─────────────────────────┘
                            │ HTTPS / REST（携带 Sa-token 令牌）
┌───────────────────────────┴─────────────────────────┐
│                  Spring Boot 后端 (单体分层)           │
│  Controller → Service → Mapper(MyBatis-Plus)          │
│  Sa-token（登录态/注解鉴权） | 全局异常 | 参数校验      │
│  多租户插件 TenantLineInnerInterceptor 自动隔离        │
│  基座 system + 业务定制 biz/*（模块化单体）            │
└───────────────────────────┬─────────────────────────┘
                            │ JDBC
┌───────────────────────────┴─────────────────────────┐
│                      PostgreSQL                       │
│   Flyway 管理 schema 版本  |  逻辑外键 + 索引          │
└───────────────────────────┬─────────────────────────┘
                            │
                       Redis（Sa-token 会话 / 缓存）
```

**架构选型：坚持模块化单体，当前不拆微服务。** 多租户上下文靠 Sa-token 会话 + MyBatis-Plus 插件**进程内透传**，单体内最简单；微服务的网关 / 服务发现 / 分布式事务是为大团队、大流量摊薄的成本，现阶段属过度设计。只要守住模块边界（业务模块经 service 接口交互、不跨模块直连 mapper），将来可按 `biz/*` 平滑切出微服务——**为拆做好准备，但不提前拆**。

## 2. 技术栈明细

### 2.1 前端
| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | Vue 3 (Composition API) | `<script setup>` 语法 |
| 构建 | Vite 5 | 快速冷启动与 HMR |
| 语言 | TypeScript | 类型安全 |
| 状态 | Pinia | 官方推荐，替代 Vuex |
| 路由 | Vue Router 4 | 路由级权限守卫 |
| UI 库 | Element Plus | 中后台组件齐全 |
| HTTP | Axios | 统一拦截器处理 token/错误 |
| 图表 | ECharts | 数据可视化（业务模块按需） |
| 工具 | dayjs / lodash-es | 日期与工具函数 |
| 代码规范 | ESLint + Prettier | 统一风格 |

### 2.2 后端
| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | Spring Boot 4.0（Spring Framework 7） | 需 JDK 17+；自动配置已模块化，集成第三方库须用 `spring-boot-starter-*` |
| Web | Spring Web (REST) | |
| 安全 | **Sa-token** | 登录认证 + 注解式角色/权限校验；会话存 Redis（`sa-token-redis-jackson`）分布式共享；不依赖 Spring Security / JWT |
| 持久层 | **MyBatis-Plus** | SQL 可控、代码生成器、内置分页与多租户插件 |
| 多租户 | **MyBatis-Plus `TenantLineInnerInterceptor`** | 自动按 `tenant_id` 改写 SQL，业务代码无感 |
| 数据库迁移 | Flyway | schema 版本化，启动时自动按版本顺序执行 SQL，不重复 |
| 校验 | Jakarta Validation | `@Valid` 参数校验 |
| 文档 | SpringDoc OpenAPI (Swagger UI) | 自动生成 API 文档 |
| 工具 | MapStruct / Lombok | Entity/DTO/VO 映射、样板代码 |
| 构建 | Maven | 默认构建工具 |

### 2.3 基础设施
- **数据库**：PostgreSQL 16+
- **缓存/会话**：Redis 7+ — 存储 Sa-token 登录会话（分布式）、热点缓存
- **容器化**：Docker + docker-compose 一键拉起 PostgreSQL + Redis + 后端 + 前端，区分开发与生产环境
- **多环境**：开发(dev) 与 生产(prod) 两套 — 后端 Spring Profiles（dev/prod）+ 前端 Vite 模式（.env.development / .env.production）。开发测试统一用 dev（跑在测试机），prod 部署到云服务器
- **定时任务**：Spring `@Scheduled` 或 Quartz（按需，可选）

## 3. 工程结构

```
project/
├── .gitignore                       # Git 忽略规则（target、node_modules、密钥等）
├── README.md                        # 项目说明与启动指南
├── backend/                         # Spring Boot
│   ├── src/main/java/com/github/lyd11250/bedrock/
│   │   ├── common/                  # 通用：响应体、异常、工具、BaseEntity
│   │   ├── config/                  # Sa-token、CORS、Swagger、多租户、审计配置
│   │   ├── system/                  # 【基座】系统能力：租户/套餐/菜单/角色/用户/鉴权（RBAC）
│   │   └── biz/                     # 【业务定制】上层业务模块，基座不内置，按需扩展
│   │       └── <module>/            #   单个业务模块：controller/.../ + 设计文档就近
│   │   每个模块下：controller / service / mapper / entity / dto / vo
│   │   （dto=接口入参，vo=接口出参，entity=持久层对象，省略 BO）
│   ├── src/main/resources/
│   │   ├── application.yml          # 公共配置
│   │   ├── application-dev.yml      # 开发（含测试，跑在测试机）
│   │   ├── application-prod.yml     # 生产（云服务器）
│   │   └── db/migration/            # Flyway SQL 脚本（建表/演进）
│   └── pom.xml
├── frontend/                        # Vue3
│   ├── src/
│   │   ├── api/                     # axios 接口封装
│   │   ├── views/                   # 页面（按模块分目录）
│   │   ├── components/              # 公共组件
│   │   ├── stores/                  # Pinia
│   │   ├── router/                  # 路由 + 守卫
│   │   ├── directives/              # 指令（v-permission 等）
│   │   ├── layouts/                 # 布局
│   │   └── utils/
│   ├── .env.development             # 开发环境变量（含测试）
│   ├── .env.production              # 生产环境变量
│   ├── vite.config.ts
│   └── package.json
├── docker-compose.yml               # 基础编排：postgres + redis + backend + frontend
├── docker-compose.dev.yml           # 开发覆盖：暴露端口、挂本地卷、热重载
└── docker-compose.prod.yml          # 生产覆盖：资源限制、健康检查、不对外暴露 DB/Redis
```

> 说明：基础包为 `com.github.lyd11250.bedrock`；`system` 为基座系统能力，`biz/*` 为上层业务模块（基座不内置）。

## 4. API 技术约定（REST）

- 统一前缀：`/api/v1`
- 统一响应体：`{ code, message, data }`（`code=0` 成功，非 0 为错误码，由全局异常处理器统一产出）
- 鉴权：除登录/注册等开放接口外，请求头携带 Sa-token 令牌（默认头名 `satoken`，可配置）
- 分页：`?page=1&size=20&sort=createdAt,desc`，返回 MyBatis-Plus 分页结构 `{ records, total, current, size }`
- 完整接口由 Swagger UI（SpringDoc OpenAPI）自动维护

## 5. 非功能性需求

- **安全**：密码 BCrypt 加盐；接口防越权；多租户严格隔离（插件按 `tenant_id` 自动过滤）。
- **性能**：列表接口分页 + 索引；高频读可加 Redis 缓存。
- **可观测**：统一日志（logback）、全局异常处理、操作审计字段（created/updated by/at）。
- **测试**：后端 JUnit + MockMvc；前端 Vitest（关键逻辑）。

## 6. 部署与环境

### 6.1 环境划分（两套）
| 环境 | 用途 | 载体 | Spring Profile | 前端模式 |
|------|------|------|----------------|----------|
| 开发 dev | 开发 + 测试 | 测试机 | dev | development |
| 生产 prod | 线上 | 云服务器 | prod | production |

### 6.2 后端多环境
- `application.yml` 放公共配置；`application-dev.yml` / `application-prod.yml` 放各环境差异（DB、Redis 地址、日志级别、Sa-token 配置）。
- 通过环境变量 `SPRING_PROFILES_ACTIVE=dev|prod` 激活。
- 生产敏感配置（DB/Redis 密码等）通过环境变量注入，不写入仓库。

### 6.3 前端多环境
- Vite 按模式加载 `.env.development` / `.env.production`，区分 `VITE_API_BASE_URL` 等。
- 生产构建：`vite build --mode production`，产物由 Nginx 容器托管。

### 6.4 一键部署
- 基础编排 `docker-compose.yml` 定义 postgres + redis + backend + frontend(nginx) 四个服务。
- 环境覆盖：`docker-compose.dev.yml`（暴露端口、挂本地卷、热重载）与 `docker-compose.prod.yml`（资源限制、健康检查、不对外暴露 DB/Redis）。
- 启动命令：
  - 测试机（开发/测试）：`docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d`
  - 云服务器（生产）：`docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d`

## 7. 关键技术决策汇总

| 维度 | 决策 |
|------|------|
| 架构风格 | 模块化单体 + 传统分层（controller/service/mapper + 贫血模型） |
| 持久层 | MyBatis-Plus |
| 建表/迁移 | Flyway 管理 SQL 脚本 |
| 鉴权 | 仅 Sa-token（不用 JWT / Spring Security），会话存 Redis |
| 多租户 | MyBatis-Plus 多租户插件，`tenant_id` 取自 Sa-token 会话，业务代码无感 |
| 分层对象 | Entity + DTO + VO（省略 BO） |
| 缓存/会话 | Redis 7+ |
| 前端 UI | Element Plus |
| 构建 | Maven（后端）/ Vite（前端） |
| 部署 | Docker Compose 一键部署，区分 dev（测试机）/ prod（云服务器） |
| 版本管理 | Git（`.gitignore` 忽略构建产物与密钥，敏感配置不入库） |
