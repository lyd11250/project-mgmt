## 一、安全

### G1 · 租户管理员可越权提权为超管 ⚠️ 真实漏洞
- **现象**：`RoleService.create` 对角色 `code` 无保留字校验；`StpInterfaceImpl` 只要角色码含 `SUPER_ADMIN` 即返回通配 `*` 并绕过套餐边界。租户管理员可在本租户新建 `code=SUPER_ADMIN` 的角色 → 分配给自己 → 取得 `*`。
- **影响**：因 `tenant` 表在 `IGNORE_TABLES`（全局、不按租户过滤），提权后可列出/操作**所有租户**，属基座级越权。
- **建议**：系统角色码（`SUPER_ADMIN` 等）设为保留字，禁止租户内创建/复用；超管判定改为绑定平台租户或独立标志位，而非仅凭角色码字符串。
- **涉及**：`system/service/RoleService.java`、`system/StpInterfaceImpl.java`、`system/RbacConstants.java`
- **优先级**：P0 ｜ **状态**：待评估

### G3 · 登录无防爆破
- **现象**：`AuthService.login` 无失败计数、锁定、验证码。
- **建议**：失败 N 次锁定/退避（Redis 计数），可选滑块/图形验证码；记录失败来源 IP。
- **涉及**：`system/service/AuthService.java`
- **优先级**：P1 ｜ **状态**：待评估

### G4 · 无密码策略
- **现象**：`create/resetPassword` 直接 `encode(raw)`，无复杂度/长度/历史/强制改密。
- **建议**：可配置的密码强度校验；首次登录/重置后强制改密；可选密码有效期。
- **涉及**：`system/service/UserService.java`、新增校验组件
- **优先级**：P1 ｜ **状态**：待评估

### G5 · 会话策略偏宽 + 状态变更不失效会话
- **现象**：`timeout=30天`、`active-timeout=-1`（永不闲置失效）；租户停用、套餐变更后既有在线会话不强制失效（权限实时跟随，但登录态仍在）。
- **建议**：缩短闲置超时/引入续期；停用租户或用户、变更套餐时主动踢出相关会话（`StpUtil.logout`/`kickout`）。
- **涉及**：`application.yml`、`system/service/TenantService.java`、`UserService.java`
- **优先级**：P1 ｜ **状态**：待评估

## 二、质量保障

### G2 · 核心机制零自动化测试 ⚠️
- **现象**：无 `src/test`；`spring-boot-starter-test` 已引未用。
- **影响**：最易错、被上层 `biz/*` 长期依赖的两块无回归兜底——①多租户行级隔离；②RBAC「角色菜单 ∩ 套餐边界」交集与超管通配。
- **建议**：补集成测试（Testcontainers PG/Redis）覆盖：跨租户数据不可见、套餐边界裁剪、超管通配、越权提权（G1）被拦截；关键 Service 单测。
- **涉及**：新增 `backend/src/test/...`、`pom.xml`（testcontainers）
- **优先级**：P0 ｜ **状态**：待评估

## 四、SaaS 核心能力

### G8 · 套餐只圈菜单边界，无配额限额 ✅
- **现象**：`sys_package` 仅关联菜单，无用户数/存储/调用量等配额，租户可无限建用户。
- **方案（已落地）**：配额采用「字典 + 引用」两表模型——`sys_quota_def(quota_key 唯一, name, remark, sort)` 定义配额项（含前端展示名称），`sys_package_quota(package_id, quota_id, quota_value)` 引用字典记录各套餐上限；`-1`/无记录=不限（缺省放行）。基座内置定义 `max_users → 最大用户数`（`QuotaKeys.MAX_USERS`），种子：基础版 10、全功能不限。通用 `QuotaService.checkAndAssert(key, intendedTotal)`（按 key 查 def → 查套餐值）在 `UserService.create` 拦截。**新增配额维度（含 biz 模块）只需向 `sys_quota_def` 注册定义 + 在写操作处调 `QuotaService`，无需改基座代码**。前端套餐页配额从表格列移除，改为操作列「配置配额」按钮 + 独立对话框（按名称展示、空=不限），权限码 `system:package:quota`。
- **演进**：初版为键值表 `sys_package_quota(quota_key, quota_value)` + 套餐 DTO 内联 `Map<String,Long> quotas`（`V4`）。因配额维度可能很多、且需中文名称展示，`V5__quota_def.sql` 重构为字典引用模型并将配额配置从套餐增改表单中拆出为独立接口（`GET/PUT /system/packages/{id}/quotas`）。
- **涉及**：Flyway `V4__package_quota.sql`/`V5__quota_def.sql`、`TenantLineHandlerImpl`(IGNORE_TABLES)、`SysQuotaDef`+Mapper、`SysPackageQuota`+Mapper、`QuotaKeys`、`QuotaService`、`UserService`、`PackageController/Service`、`AssignQuotasDTO`/`PackageQuotaVO`、前端 `package.ts`/`PackageList.vue`
- **优先级**：P1 ｜ **状态**：已完成

### G11 · 无订阅生命周期
- **现象**：`tenant.package_id` 永久绑定，无有效期/到期/计费/续费。
- **已落地**：
  - 接通悬空的 `tenant.expire_at`——`TenantCreateDTO/Service/VO` 支持设置/展示到期时间。
  - **续费 + 套餐变更**：`PUT /system/tenants/{id}/renew`（超管）覆盖到期时间（留空=永久，用 update wrapper 显式 set 以落库 null），可选 `changePackageId` 同时升/降套餐（套餐边界由 `StpInterfaceImpl` 每次鉴权实时读 `package_id`，变更即生效，无需踢会话）。前端「续费」对话框含套餐下拉。
  - **惰性过期判定（不落库、无定时任务）**：`TenantService.isExpired(tenant)` 共享判定逻辑（平台租户 + 永久豁免）。请求级 `TenantStatusInterceptor`（注册于登录拦截器之后）对已登录会话校验「租户存在 / 停用 / 过期」，过期或停用即拦（403），续费后下次请求自动恢复。`AuthService.login` 复用同一判定。**只读不写**——不修改 `tenant.status`，避免审计污染与状态漂移。
  - **过期展示**：`TenantVO.expired` 派生字段（现算），前端列表与续费框「已过期」红标。
- **剩余（仍 P2）**：独立订阅记录（起止/状态/历史）、临期预警、计费对接（计费建议移交上层 `biz/billing`，基座不实现）。
- **涉及**：`Tenant`/`TenantCreateDTO`/`TenantRenewDTO`/`TenantController`/`TenantService`/`TenantVO`/`AuthService`/`TenantStatusInterceptor`/`SaTokenConfig`、前端 `tenant.ts`/`TenantList.vue`
- **优先级**：P2 ｜ **状态**：部分完成

### G10 · 鉴权多次查库且无缓存

## 七、工程细节 / 架构

### G9 · API 文档缺失 ✅
- **现象**：springdoc/Swagger 在 `pom.xml` 中被显式注释推迟（Boot4 兼容版本待定）。
- **已落地**：引入 `springdoc-openapi-starter-webmvc-ui` `3.0.3`（Spring Boot 4 / Spring Framework 7 兼容线，Jackson 3）。`config/OpenApiConfig` 配置元信息 + Sa-token 鉴权方案（apiKey + header，头名取 `sa-token.token-name`），各 Controller 加 `@Tag` 归类；Swagger UI `/swagger-ui.html`、JSON `/v3/api-docs`，`SaTokenConfig.EXCLUDE_PATHS` 放行文档资源；dev 开启、prod 关闭（`application-prod.yml`）。**同步统一 API 路径**为 `/api/v1/{module}/{resource}`（系统能力收敛至 `/api/v1/system/*`，含 auth；ping 保持根级），约定写入 [CLAUDE.md](CLAUDE.md) 4.8。
