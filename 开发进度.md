## 阶段明细

### ✅ 第 1 期-A：认证权限基座
**后端**
- `auth` 域：登录 / 登出 / `me`，用户·角色·租户三套 CRUD。

- 多租户隔离（`TenantLineHandlerImpl` + `TenantContext`）。
- 通用响应 / 全局异常 / 审计自动填充。
- Flyway `V1__init_core.sql`（tenant / sys_user / sys_role / sys_permission / 关联表）。

**前端**
- 登录页、默认布局、用户管理、租户管理页。
- api / Pinia store / 路由守卫 / `v-permission` 指令。

### ✅ 第 1 期-A2：RBAC 套餐化重构 + 重命名为 Bedrock
目标：把简化 RBAC 升级为「全局菜单目录 + 套餐订阅 + 租户内 RBAC」三层模型。设计见 [功能设计.md](功能设计.md) 第 2、4 节。分两个提交落地（重命名 / 套餐化重构）——

- [x] **Flyway `V2__rbac_package.sql`**：建 `sys_menu`（全局）、`sys_package`、`sys_package_menu`、`sys_role_menu`；`tenant` 加 `package_id`；删 `sys_permission` / `sys_role_permission`；初始化菜单种子 + 默认套餐（BASIC / FULL）。
- [x] **多租户**：`sys_menu` / `sys_package` / `sys_package_menu` 加入 `IGNORE_TABLES`（全局表不带 tenant_id）；全局实体用 `common.GlobalBaseEntity`（无 tenantId）。
- [x] **重命名为 Bedrock**：基础包 `…project` → `…bedrock`；`auth` 域并入统一 `system` 模块；主类 `BedrockApplication`；构件 `saas-bedrock`（jar / `spring.application.name` / Dockerfile / DB 默认值同步）。
- [x] **实体/CRUD**：新增 `SysMenu` / `SysPackage` 及菜单管理、套餐管理（仅超管）；角色管理升级为「角色↔菜单分配」（限套餐边界）。
- [x] **鉴权重写**：`StpInterfaceImpl` 改为 角色→菜单 perm ∩ 套餐边界；超管通配 `*`；权限码对齐 `system:资源:动作`。
- [x] **种子重写**：`SeedService.seedTenant(tenantId, packageId, …)` 绑套餐 + 只播角色/管理员；`RbacConstants` 移除硬编码权限目录，权限码迁至 DB。
- [x] **前端动态菜单**：`/system/menus/nav` 下发菜单 → 动态导航（`SideMenu`）+ 动态路由（`import.meta.glob`），替代写死的 `router/index.ts`；新增菜单/套餐/角色管理页；租户创建选套餐。
- [x] **自测**：本机 `mvn -s settings.xml compile` + `npm run build` 通过；测试机三视角（超管/租户管理员/普通用户）验证待执行。

### 🔧 基座增强（A2 后，来自[待完善清单](待完善清单.md)）

A2 交付后，按待完善清单择项增强基座能力，每项完成后在清单内标 `已完成`，并在此登记：

- [x] **G1 越权提权修复**：系统角色码设为保留字，超管判定不再仅凭角色码字符串。
- [x] **G8 套餐配额**：
  - 初版（`V4__package_quota.sql`）：键值配额表 `sys_package_quota(quota_key, quota_value)`，内置 `max_users`，`QuotaService` 在 `UserService.create` 拦截。
  - 重构（`V5__quota_def.sql`）：改为「字典 + 引用」两表模型——`sys_quota_def`（quota_key + 展示名称）+ `sys_package_quota(package_id, quota_id, quota_value)`；配额配置从套餐增改表单拆为独立接口（`GET/PUT /system/packages/{id}/quotas`）与「配置配额」对话框，权限码 `system:package:quota`。便于后续业务自定义大量配额维度并友好展示名称。
- [x] **G11 订阅到期（部分）**：接通 `tenant.expire_at`，过期拒登（平台租户豁免）；续费接口 `PUT /system/tenants/{id}/renew`（超管，覆盖到期时间[留空=永久] + 可选变更套餐）+ 前端「续费」对话框（含套餐下拉）。新增请求级 `TenantStatusInterceptor` 惰性校验「停用/过期」（只读不写、不改 status、无定时任务，续费后自动恢复），与登录共用 `TenantService.isExpired`；`TenantVO.expired` 派生字段 + 前端「已过期」标签。
- [x] **G9 API 文档 + 统一 API 格式**：引入 springdoc-openapi `3.0.x`（Boot4 兼容），Swagger UI `/swagger-ui.html`（dev 开 / prod 关），Sa-token 鉴权方案 + `@Tag` 归类；统一 API 路径为 `/api/v1/{module}/{resource}`——系统能力收敛至 `/api/v1/system/*`（auth 一并迁入，ping 保持根级），前后端同步，约定写入 [CLAUDE.md](CLAUDE.md) 4.8。
- [x] **G4 密码强度校验（初步）**：自定义 `@StrongPassword`（长度 8-64 + 字母数字混合 + 无空白），挂建用户/管理员重置/自助改密三处；规则常量集中、留配置化扩展位。密码历史/有效期/强制改密仍待排期。
- [x] **G12 自助能力（改密 + 个人中心）**：`sys_user` 加 `nickname`/`phone`（`V7`）；新增 `GET/PUT /system/users/me/profile`、`PUT /system/users/me/password`（登录即可）；个人中心页（资料 + 改密两 Tab）。注册/找回密码/验证码不做（无邮件短信基础设施）。
- [x] **G16 前端完善**：错误页 403/404/500 + catch-all 兜底 + 守卫跳 403 + 动态路由占位兜底；顶栏改圆形默认头像（Element `Avatar`）下拉菜单「个人中心/修改密码/退出登录」。
- [x] **文件上传下载（MinIO 对象存储）**：
  - 存储抽象 `StorageProvider` + MinIO 实现（`MinioStorageProvider`，对象键按 `租户/年月` 分桶、单遍计算 SHA-256、启动自动建桶），切换后端不动业务代码。
  - Flyway `V10__file_storage.sql`：`sys_file` 元数据表（带 `tenant_id` 走多租户自动隔离）；新增存储配额 `max_storage_bytes`（基础版 1GiB / 全功能不限，复用 `QuotaService`）；文件管理菜单 + `system:file:*` 权限种子并纳入全功能套餐。
  - 接口：上传（白名单 + 配额校验）/ 流式下载（RFC 5987 中文文件名，不走统一 `Result` 包装）/ 分页 / 软删（物理对象保留，待后续定时任务清理）。
  - 前端：文件管理页 `FileList` + `el-upload` 上传 + blob 下载（独立裸 axios 实例携带令牌）。
  - 编排：`docker-compose` 增加 `minio` 服务（固定稳定版 `RELEASE.2025-09-07T16-13-09Z`），注入端点/凭据/桶名，dev 暴露 9000/9001；nginx `client_max_body_size 50m` 与后端 multipart 上限对齐（修复大文件 413）。
  - 定位收敛：文件管理台为「管理员运维入口」，访问完全交给 RBAC 分发（`system:file:*` 由管理员按角色授予）；`SeedService` 调整为普通用户（USER 角色）默认不预绑文件管理页（用 perm 排除，租户管理员仍可见）。
- [x] **用户头像模块**（复用文件存储）：
  - `sys_user` 加 `avatar_file_id`（`V11__user_avatar.sql`）；头像复用 `FileService` 落 `sys_file`（计入 `max_storage_bytes` 配额），换图软删旧记录。
  - 鉴权定位：上传走「登录即可」（`PUT /system/users/me/avatar`，**不复用** `system:file:upload`）；读取「本租户登录可见」（`GET /system/users/{id}/avatar`，内联流）。生产 MinIO 不对外暴露，头像经后端代理（前端带令牌拉 blob → objectURL → `<img>`）显示。
  - 前端：个人中心头像上传区（限图片 ≤2MB）、顶栏头像、用户管理页列表头像列；`MeVO`/`ProfileVO`/`UserVO` 补 `avatarFileId`，auth store 缓存并在登出/换图时释放 objectURL。
- **自测**：本机 `mvn -s settings.xml compile` + 前端 `vue-tsc` 通过；本机无 Docker，迁移与页面交互需测试机实跑验证。
